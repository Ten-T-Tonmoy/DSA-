## **1️⃣ Helper Functions**

### `applyOp(int a, int b, char op)`

* Applies a basic arithmetic operator (`+`, `-`, `*`, `/`) on two operands `a` and `b`.
* Example: `applyOp(2,3,'*') → 6`.

### `priority(char op)`

* Returns the **precedence of operators**:

  * `*` and `/` → 2
  * `+` and `-` → 1
  * others → 0
* Used for **infix evaluation** to decide when to apply operators.

---

## **2️⃣ Infix Evaluation: `solveInfix()`**

- Uses **two stacks**:

  1. `values` → stores numbers
  2. `ops` → stores operators and parentheses

### **Mechanism**

1. **Scan left → right**.
2. **Digits** → extract multi-digit number → push onto `values`.
3. **'('** → push to `ops`.
4. **')'** → pop operators from `ops` and apply on `values` until `'('` is found.
5. **Operator (+,-,\*,/)** → pop and apply any operators from `ops` with **higher or equal precedence**, then push the current operator.
6. **At the end** → apply remaining operators in `ops`.

---

**Example:** `"((2+3)*5)"`

- Push `(`
- Push `(`
- Push `2`
- Operator `+` → push `+`
- Push `3`
- `)` → pop `+`, compute `2+3=5` → push 5
- `*` → push `*`
- Push `5`
- `)` → pop `*`, compute `5*5=25` → push 25

✅ Result: 25

---

## **3️⃣ Postfix Evaluation: `solvePostfix()`**

- Uses **one stack**:

  - Push operands
  - On operator: pop **two operands**, apply operator, push result back

**Mechanism**

1. Scan **left → right**.
2. Digit → push.
3. Operator → pop `b`, pop `a`, compute `a op b`, push result.

**Example:** `"23+5*"`

- `2` → push 2
- `3` → push 3
- `+` → pop 3 & 2 → 5 → push 5
- `5` → push 5
- `*` → pop 5 & 5 → 25 → push 25

✅ Result: 25

---

## **4️⃣ Prefix Evaluation: `solvePrefix()`**

- Uses **one stack**:

  - Scan **right → left** (because prefix operator comes before operands)
  - Push operands
  - On operator → pop **two operands**, apply, push result

**Mechanism**

1. Right → left scan
2. Digit → push
3. Operator → pop `a` & `b` → compute `a op b` → push

**Example:** `"+34"`

- `4` → push 4
- `3` → push 3
- `+` → pop 3 & 4 → 3+4=7 → push 7

✅ Result: 7

> Notice: prefix evaluation **doesn’t reverse the operator**, it pops operands in correct order for computation.

## **5️⃣ `evaluate(string expr)`**

- Determines expression type:

```cpp
if (expr[0] is operator) → prefix
else if (expr.back() is not digit) → postfix
else → infix
```

- Then calls the respective solver.

**Example tests from your code:**

| Test | Expression    | Solver  | Result |
| ---- | ------------- | ------- | ------ |
| 1    | `"23+5*"`     | Postfix | 25     |
| 2    | `"+34"`       | Prefix  | 7      |
| 3    | `"2+3*5"`     | Infix   | 17     |
| 4    | `"84/"`       | Postfix | 2      |
| 5    | `"((2+3)*5)"` | Infix   | 25     |

---

### **Summary Table**

| Expression Type | Scan Direction | Stack Usage | Pop Order / Apply     |
| --------------- | -------------- | ----------- | --------------------- |
| Infix           | Left → Right   | values, ops | Use precedence        |
| Postfix         | Left → Right   | values      | Pop b then a → a op b |
| Prefix          | Right → Left   | values      | Pop a then b → a op b |
